## Structure convention

Every hub follows:
1. TL;DR comment (one line, for quick scan)
2. **Current Status** (3-8 lines — what Claude reads first via pre-flight check)
3. **Open Gaps** (structured unknowns with Status/Question/Tried/Closes when)
4. Detail sections (as needed)
5. **Changelog** (append-only, at bottom)

Non-obvious facts carry provenance dates `[YYYY-MM]`.
