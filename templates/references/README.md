# References

Deep research artifacts. Load on-demand only when the topic comes up.

## What belongs here

- Frameworks extracted from books (e.g., decision-making models, habit frameworks)
- Research summaries with structured takeaways
- Domain-specific reference material (e.g., industry metrics, technical specs)

## Convention

Each reference file should include:
- **Source**: Book/paper/article title and author
- **Key concepts**: Structured extraction, not a summary
- **Application rules**: When and how Claude should use this framework
- **Provenance date**: When this was extracted

## Example references you might add

- Decision-making frameworks (e.g., Edmans' Ladder of Misinference)
- Habit formation models (e.g., Atomic Habits, Tiny Habits)
- Industry-specific knowledge bases
- Negotiation or communication frameworks
